Главное меню (Навигация)

Сделайте приложение с боковым меню (Sidebar) или вкладками (Tabs) со следующими разделами:

    👥 Сотрудники и Кадры

    📁 Проекты и Заказчики

    🐞 Тестирование и Баги

    💰 Финансы и Инфраструктура

    📊 Отчеты и Документы

Давайте разберем, что должно быть на каждом экране и какие функции там вызывать.
Экран 1: 👥 Сотрудники и Кадры

Для чего: Поиск нужных специалистов, оценка их загрузки.

    Верхняя часть (Фильтры):

        Выпадающий список «Отдел» (department_ids)

        Текстовое поле «Должность» (position_titles)

        Два поля ввода «Стаж от... до...» (p_min_experience, p_max_experience)

        Кнопка «Найти»

    Нижняя часть (Таблица):

        Сюда выводится результат функции get_employee_by_filters(...).

    Доп. кнопка «Посмотреть загрузку»:

        Выбираете сотрудника в таблице, нажимаете кнопку -> открывается модальное окно. В нем вызывается get_employee_workload(employee_id), которая показывает, на каких проектах он сейчас работает и сколько часов списывает.

Экран 2: 📁 Проекты и Заказчики (Основной рабочий экран)

Для чего: Управление портфелем проектов.

    Вкладка «Проекты»:

        Фильтры: Статус проекта (в разработке, завершен), Бюджет от/до, Методология.

        Таблица: Результат get_projects_by_filters(...).

        Интерактив: При двойном клике на строку проекта открывается «Карточка проекта» (см. ниже).

    Вкладка «Заказчики»:

        Фильтры: Период дат.

        Таблица: Результат get_client_activity(from, to). Показывает, сколько денег принес клиент.

🌟 Карточка проекта (Открывается при клике на проект)

Это окно с несколькими внутренними вкладками, которое собирает данные по одному конкретному project_id:

    Команда: Вызов get_project_team(project_id). Показывает, кто работает над проектом.

    Тех. Задание: Делаем простой SELECT * FROM ProjectSpecification WHERE project_id = X ORDER BY version DESC. Здесь можно показать текстовое поле с ТЗ (которое сгенерировал наш триггер!) и дать кнопку "Сохранить", если аналитик внес туда ручные правки.

    Статус-отчет: Вызов функции get_project_status_report(project_id, start, end). Выводит плашку: текущий этап, сколько задач закрыто, сколько часов сожжено, сколько активных рисков.

Экран 3: 🐞 Тестирование и Релизы

Для чего: Оценка качества ПО.

    Верхняя часть: Выбор проекта из списка + фильтры дат.

    Таблица 1 (Баги): Вызов get_bugs_by_filter(). Показывает все ошибки проекта. Можно отфильтровать только "критические" или только "новые".

    Метрика (Эффективность QA): Вызов get_testing_efficiency(). Можно вывести в виде красивой диаграммы (Круговая диаграмма: % багов найденных в тесте vs % найденных на проде).

    Таблица 2 (Релизы): Вызов get_release_report(). Показывает историю версий продукта.

Экран 4: 💰 Финансы и Инфраструктура

Для чего: Для руководителей и бухгалтерии.

    Вкладка «Рентабельность»: Вызов get_project_profitability(). Выводите в таблицу проекты, где можно подсвечивать efficiency_ratio цветом (Зеленый = прибыльно, Красный = вышли за бюджет).

    Вкладка «Лицензии и Серверы»: Вызов get_infrastructure_report(). Таблица, где красным подсвечиваются лицензии, срок действия которых (expiry_date) скоро истекает.

    Вкладка «Ежемесячный отчет»: Пользователь выбирает Месяц и Год. Вызывается get_monthly_financial_report(month, year). Идеально для вывода гистограммы (столбчатой диаграммы): Доходы vs Зарплаты vs Серверы по каждому проекту.
